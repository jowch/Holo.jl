## The payload terms

A rendered `:webgl` cell ships three payloads over Pluto's `published_to_js` (MsgPack on the wire);
the click *return* value (`{layer,index,payload}`) is tiny and not a factor:

| Term | Carried by | Cost driver | Shipped |
|------|-----------|-------------|---------|
| **WGLMakie bundle** | `published_to_js` в†’ blob URL в†’ `import()` | fixed (vendored bundle + three.js + atlas) | **once per notebook** (M2) |
| **scene** | `published_to_js` (MsgPack binary) | #plots Г— geometry + glyph atlas | every render (per cell / per frame) |
| **manifest + overlay** | reuses Holo core verbatim | #hit-elements | every render (tiny вЂ” see `../../docs/perf-findings.md`) |

The **bundle dominated** and is now shared once per notebook (M2 / PR #18), so the per-cell cost is
just the **scene**.

### Stress + server cost (the terms the head-to-head turns on)

Two `:webgl` facts under stress (measured by `bench/vs_cairo.jl`, same seed as the envelope):

| scene | wire | serialize ms |
|---|---|---|
| scatter 100 000 | 0.86 MB | ~30 |
| heatmap 200ВІ | 1.96 MB | ~30 |
| heatmap 500ВІ | 11.8 MB | ~45 |

- **Serialize time is ~flat (~30 ms) regardless of N** вЂ” `scene_payload` only *serializes*; the GPU
  draw is offloaded to the client. (Contrast Cairo, which rasterizes server-side: its render+encode
  climbs to ~2.2 s at 100k points вЂ” `../../docs/perf-findings.md` / `bench/stress.jl`.) This flat line
  is the per-update UX win, not a measurement gap.
- **Dense rasters blow up the scene**: a heatmap ships as a value grid, so 500ВІ = 11.8 MB vs a ~1 MB
  Cairo PNG. `:webgl` is *not* universally lighter вЂ” rasters are Cairo's home turf.

**Cross-backend comparison (wire + UX + capability matrix):** see
[`backend-comparison.md`](backend-comparison.md), generated by `bench/vs_cairo.jl` вЂ” the head-to-head,
the crossover-N analysis, and the interaction capability matrix (pan/zoom/rotate are client-local on
`:webgl`, impossible on Cairo's static PNG). It reconciles with this file and root's envelope.

### Wire vs JSON proxy

`published_to_js` does **not** ship the scene as JSON text. Pluto's MsgPack encodes every typed
numeric `Vector` (`Float32`/`Int32`/`UInt32`/`UInt8` вЂ” exactly what `_plain` emits) as a **binary**
extension (`reinterpret(UInt8, x)`, `sizeofВ·length`). So the real wire is the **binary** column,
**~4вЂ“5Г— under** `JSON3.write` (floats-as-text) вЂ” the proxy an earlier bench reported. The committed
bench reports both; the `wire` column is what actually crosses the wire (dominant term; structural
map/string overhead adds a little).

## Bundle sharing вЂ” why it's once per notebook (M2 / PR #18)

`published_to_js` ids are content-addressed (`notebook_id/objectid(x)`, and `objectid(::String)` is
content-based), so the one `Ref`-cached bundle string has a **stable id** that crosses the wire
exactly once: across cells, Pluto's notebook merge keeps one copy on load; across re-runs of a cell,
Pluto nulls already-known ids before sending (`known_published_objects` + `format_output.jl`), so a
re-run re-ships only its new-id scene, never the stable-id bundle. The browser then caches the
bundle/shim blob URLs once on `window.__HoloWGL` so the WGLMakie module imports once, not per cell.

## Deferred compression levers (measured, not yet built)

The per-cell scene is already small (binary), so compression is **deferred** вЂ” both levers measured:

- **gzip.** The bench's `gzip-bin` column measures gzip-of-binary at **~3Г—** (0.07в†’0.02 MB), but to
  use it we'd have to bypass `published_to_js`'s object channel and hand-roll a **msgpack decoder in
  JS**. The cheap path вЂ” gzip-of-JSON via the browser's native `DecompressionStream` в†’ `JSON.parse`
  (the `gzip-json` column) вЂ” buys only **~25%** vs the current wire, since it starts from float-text.
  Not worth a new JS decoder + failure surface for ~0.05 MB/cell yet.
- **Atlas sharing.** The glyph-atlas tiles (`glyph_data/atlas_updates/<hash>`) carry content hashes
  **observed to repeat across scenes** (the digit/label tiles recur in all three bench figures above),
  so they're shareable like the bundle вЂ” but each is ~10вЂ“20 KB, gzip overlaps the win, and hoisting
  them to a shared channel is real complexity.

bottleneck** вЂ” tier-2 in-place patching (NOTES.md) already ships no new scene at all.
